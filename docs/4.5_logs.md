# Hypothesis 4.5 Development Log

This file is a simple historical summary of how Hypothesis 4.5 changed during discussion and implementation. 

## 1. Original Hypothesis 4.5

ANewOmni represents a complete peptide as several related latent blocks. Generating every block independently may produce individually plausible particles that form an invalid complete object.

The original proposed solution was:

- select several complete, compatible source complexes;
- preserve correspondence between their ordered residue slots;
- choose one barycentric coefficient vector, called lambda;
- apply that same lambda to every corresponding slot;
- initialize all slots together at the terminal EFS distribution;
- replay each slot through ordinary EFS without adding springs, learned selectors, or corrective forces.

The shared lambda was meant to keep all parts of the generated object at the same interpolation location between the same complete sources. It only couples initialization. It does not guarantee that the independent backward trajectories will remain coherent.

The original project kept two models open:

1. block EFS, with one EFS system for each known ordered slot;
2. joint-vector EFS, with the complete object flattened and treated as one particle.

The real decision was supposed to come from controlled evidence, not intuition.

## 2. First implementation: too much before the base test

The first research harness introduced synthetic conformational families, orthogonal transformations, retrieval descriptors, several metrics, and shared-versus-independent lambda controls.

This was useful as an extended experiment, but it skipped the simplest question. The base shared-lambda mechanism had not yet been isolated, and the synthetic family construction made the result difficult to understand.

Decision:

- remove rotation families and other structured synthetic assumptions from the basic experiment;
- postpone ANewOmni integration;
- test the mechanism using simple NumPy code and visible trajectories first.

## 3. Clarifying what a source and a particle mean

An early misunderstanding treated a source too much like one EFS point.

The corrected definition was:

- one particle is one point in the EFS field;
- one source is a complete object containing several particles;
- all sources contain the same number of particles;
- source ownership is metadata declared before EFS;
- EFS forces still act between ordinary particles, not between source objects.

For several selected sources, particle 1 from every source is mixed to create generated particle 1, particle 2 creates generated particle 2, and so on. The same source-level lambda must be used for all generated particles.

## 4. Separate slot planes versus one pooled plane

One prototype used a different EFS plane for every particle position. This followed the ordered-slot interpretation of the original biological hypothesis.

That model was rejected for the current toy experiment because the synthetic particles have no real slot identity. Treating the first random particle of every source as a special slot would create structure that does not exist in the data.

Current toy decision:

- use one and only one pooled EFS plane;
- let every memory particle interact through the same vanilla EFS field;
- treat source membership only as information used when selecting and interpolating complete sources;
- do not give particle positions separate centroids, distributions, or EFS systems.

This is simpler than the original ordered-slot H4.5 model. It tests the shared-source and shared-lambda principle before any biological slot structure is introduced.

## 5. Increasing scale and improving observability

The experiment was expanded from small fields to thousands of particles. Forward and backward logs were added to show:

- average particle distance;
- field radius and contraction;
- force and update size;
- complete-source position and displacement during replay;
- whether backward movement becomes stuck, unstable, or explosive.

Two-dimensional and PCA flow plots were added to make the complete path visible from the initial distribution, through forward EFS, terminal interpolation, and backward replay.

The implementation remained CPU-only and used the exact pooled particle interaction rather than introducing an approximate model.

## 6. Calibration and diagnostic changes

A hyperparameter search was introduced because individual hand-picked settings were not enough to judge EFS behavior.

The search considered forward geometry, numerical stability, and vertex replay before examining target recovery.

The first diagnostic rules were too strict. In particular, a radial CDF gap above a fixed value was initially treated as an automatic failure. This was corrected because a finite terminal distribution can be visibly shell-like while the backward inverse still works accurately.

Final diagnostic interpretation:

- nonfinite values, dimensional collapse, extreme scale changes, and failed vertex replay are hard failures;
- radial and angular measurements describe how close the terminal field is to the expected uniform geometry;
- radial mismatch alone does not invalidate backward replay;
- vertex replay only tests whether known terminal memory particles return to their initial positions;
- a near-zero vertex error does not prove that a new interpolated source is valid.

## 7. What the early results established

The experiments showed that:

- a single pooled EFS field could contract cleanly without obvious explosion;
- backward trajectories could remain smooth and stable;
- exact memory vertices could replay with very small error;
- shared lambda often performed better than an independent-lambda falsification control in deliberately barycentric synthetic targets.

They did not establish that H4.5 would work on real ANewOmni latents.

The deliberately constructed barycentric targets favored the method being tested. Later held-out reconstruction runs were mixed or negative, and one important run fitted lambda in the wrong coordinate frame. Those generation results therefore could not settle the intended hypothesis.

## 8. Rethinking target evaluation

Creating a target directly from a known shared lambda was rejected as the main evaluation because it builds the answer into the data.

The replacement idea was to use a real held-out source:

- choose one complete source as the target;
- find nearby complete sources;
- use the target only as an evaluation witness;
- fit a shared lambda from the neighboring sources;
- replay the resulting terminal seed;
- compare the arrival with the held-out target.

This still does not measure decoded molecular validity. It measures whether the EFS and shared-lambda mechanism can reconstruct the distribution-level position of a complete source.

## 9. Making the synthetic data deliberately unstructured

Structured Gaussian families, rotations, chains, and source-specific shapes were removed from the current toy data.

The chosen synthetic model now follows these rules:

- draw all particles independently from one chaotic global distribution;
- randomly assign particles to sources only after drawing them;
- give every source the same particle count;
- assume no natural particle order or source-specific region;
- compare complete sources using an explicit temporary particle matching.

This prevents the generator from succeeding merely because the data already contains the same clean shared-barycentric structure as the hypothesis.

## 10. Ideas rejected, deferred, or demoted

The following ideas were removed from the primary toy experiment or deferred:

- rotation and conformational families before the base mechanism was understood;
- a separate EFS plane for every arbitrary synthetic particle position;
- independent per-particle lambda as the intended generator;
- a learned selector inside backward EFS;
- springs, hard projection, pinning, or other forces hidden inside vanilla EFS;
- treating a radial CDF guide as a formal pass/fail test;
- claiming novelty only when an output is far from every training source;
- claiming molecular or source-level validity before decoding;
- using a perfectly constructed barycentric target as decisive evidence;
- integrating the ANewOmni checkpoint before the CPU mechanism test is credible.

Independent lambda remains useful only as a falsification control. Retrieval-only and direct interpolation remain necessary baselines.

## 11. Disagreement

The latest discussion identified three similar-looking but logically different experiments. Considering $d1$ being the initial distribution of data and $du$ being the distribution after the EFS forward.

### Comparison table

| Build sources at $d_1$  | Random particle grouping                        | Random particle grouping                              | Random particle grouping                                                                 |

| Choose $K$ neighbors    | At $d_1$                                        | At $d_u$                                              | At $d_1$                                                                                 |

| Short summary           | Right neighbors, wrong lambda frame             | Right lambda frame, wrong neighbor-selection frame    | Neighbor identity comes from real-data geometry; lambda comes from terminal EFS geometry |

Here, $d_1$ means the original data distribution and $d_u$ means the terminal EFS distribution, which should be approximately uniform after a successful forward pass.

### Previous implementation

The target and neighboring sources were chosen in the initial distribution. Lambda was also fitted there. The same lambda was later applied to the terminal positions of those parents.

This tested whether an interpolation defined before EFS survives the nonlinear forward and backward process. It did not test terminal shared-lambda initialization.

### Latest implementation

The target was removed from memory and transported passively. Both the neighboring sources and lambda were selected at the terminal distribution.

This fitted lambda in the correct terminal coordinates, but it allowed EFS to redefine which sources count as neighbors. It no longer preserved the meaning of similarity from the original data distribution.

### Intended implementation

The intended protocol is:

1. Build complete sources at the initial distribution.
2. Select one target source.
3. Find its closest complete sources at the initial distribution.
4. Save those source identities and their particle correspondence.
5. Run forward EFS.
6. Retrieve the terminal positions of those exact saved parents.
7. Fit one shared lambda jointly over every particle and coordinate of the complete source.
8. Construct the terminal seed only from the weighted terminal parents.
9. Remove the target particles from every replay frame.
10. Replay the seed and compare its arrival with the original target.

The key principle is simple: similarity is defined where the real data lives, while interpolation coefficients are defined where EFS expects a new seed.

## 12. Remaining cautions

The intended experiment is coherent, but it still has limitations:

- four parents will rarely reproduce a high-dimensional complete target exactly;
- the terminal seed must remain the weighted parent interpolation and must not be replaced with the exact target;
- removing the target before backward replay changes the field, so forward and backward are no longer exact inverses;
- particle correspondence must be saved at the initial distribution and tracked through EFS;
- the hidden target is used to fit lambda, so this is a reconstruction diagnostic rather than a deployable generation rule;
- synthetic source recovery cannot establish decoded molecular validity.

Three replay controls can separate the main sources of error:

1. replay the exact terminal target with the target retained, measuring ordinary inverse error;
2. replay the exact terminal target after removing it, measuring the removal penalty;
3. replay the shared-lambda seed after removing the target, measuring the complete H4.5 procedure.

## 13. Current conclusion

The project has not yet validated H4.5. It has shown that vanilla single-plane EFS can be numerically stable and accurately invert known particles on the synthetic CPU experiments.

The next meaningful test is no longer another broad synthetic extension. It is the narrowly defined intended protocol in the final table: select neighbors at $d_1$, keep their identities fixed, fit one complete-source lambda at $d_u$, remove the target consistently during replay, and separate inverse, removal, and interpolation errors.
